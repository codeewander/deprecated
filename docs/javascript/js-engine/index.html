JS 引擎

JS 引擎如何運行#

  • JS 引擎通常由瀏覽器供應商創建，負責將開發者撰寫的 Javascript 程式碼進行解析和編譯成電腦讀得懂的程式碼，然後運行
  • 有許多不同的 JS 引擎(JS 引擎列表)，例如： Chrome 的 V8 引擎
  • 所有引擎均遵守 ECMA Script 標準

JS 引擎(V8 為例)運作流程如下：黃色區塊為程式碼執行時期

Step 1. 解析(Parser)#

將 JS 裡的標點符號、詞彙一一解析出來，例如：var a = 2; 就會解析 var、a、=、2、;

Step 2. 抽象結構樹 AST (Abstract Syntax Tree)#

將整個原始碼的結構定義出來，將這些單元（ var、a、=、2、; ）組成抽象語法樹

Step 3. 兩種方式執行 JS 程式碼#

Interpreter 直譯器 逐行讀取程式碼，將 AST 轉換為 Bytecode，並立即執行。

特色：可以立即執行程式碼，但不會進行優化

Compiler 編譯器 在執行前將程式碼轉換成 Machine code 或較底層的語言，例如：Babel 就是 JS 的 Complier

特色：需花費時間編譯程式碼，但會優化執行的程式碼

Memory Heap & Call Stack#

JS 引擎做了許多工作，其中最大的工作是讀取和執行程式碼，因此，需要一個儲存和寫入數據的地方(Memory Heap)和紀錄追蹤程式碼執行狀況的地方(Call Stack)。

  • Memory Heap: 根據需要分配、使用和刪除記憶體，可視為無順序的儲存空間
// tell the memory heap to allocate memory for a number
const number = 11;
// allocate memory for a string
const string = "some text";
// allocate memory for an object and it's values
const person = {
first: "Kira",
last: "Yang",
};

JS 會自動進行 garbage collection，也就是函式內的變數在函式執行完後，會自動從 Memory Heap 中刪除

  • Call Stack:JS 是單線程(single thread)，一次只能執行一件事情， Call Stack 可以紀錄追蹤執行的程式碼
function multiply(a, b) {
return a * b;
}
function printSquare(n) {
let s = multiply(n, n);
console.log(s);
}
printSquare(5);

首先，printSquare 被呼叫，因此進入 stack 的最上方；在 printSquare 中會呼叫 multiply ，因此 multiply 進入 stack 的最上方；執行完後回傳計算結果，最後 console.log 印出值。

Stack 能容納的堆疊是有上限的，倘若發生無限迴圈等情況，則會造成 stack overflow
