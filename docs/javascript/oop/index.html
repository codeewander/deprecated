物件導向編程(OOP)

JS 是結合物件導向編程(Object Oriented Programming) 和函式編程(Functional Programming)的語言，其中，物件導向基於 JS原型鍊繼承(Prototype Inheritance)的特性。

從案例來了解物件導向#

假設今天要創建不同的精靈，每個角色有不同的名字和武器，可以這樣寫：

const elf = {
name: "Orwell",
weapon: "bow",
attack() {
return "acctack with" + elf.weapon;
},
};
const elf2 = {
name: "Sally",
weapon: "fire",
attack() {
return "acctack with" + elf.weapon;
},
};
elf.attack();
elf2.attack();

但是，如果創建上百個精靈，這時候可能就需要複製多段程式碼，並修改裡面的值，有點麻煩。這時候，可以利用函式創建物件的方式進行優化：

//factory functions
function createElf(name, weapon) {
return {
name: name,
weapon: weapon,
attack() {
return "attack with" + weapon;
},
};
}
const orwell = createElf("Orwell", "bow");
orwell.attack();
const sally = createElf("Sally", "fire");
sally.attack();

然而，這樣的寫法非常佔記憶體空間，且每個創建出來的精靈物件的 attack 方法是一樣的，重複存在在多個物件中。這時候，或許可以將 attack 方法抽出來變成共用方法。

const elfFunctions = {
attack() {
return "attack with" + this.weapon;
},
};
function createElf(name, weapon) {
return {
name: name,
weapon: weapon,
};
}
const orwell = createElf("Orwell", "bow");
orwell.attack = elfFunctions.attack;
orwell.attack();
const sally = createElf("Sally", "fire");
sally.attack = elfFunctions.attack;
sally.attack();

利用 Object.create 方法可以讓程式碼更簡潔

//Object.create()
const elfFunctions = {
attack() {
return "attack with" + this.weapon;
},
};
function createElf(name, weapon) {
let newElf = Object.create(elfFunctions);
newElf.name = name;
newElf.weapon = weapon;
return newElf;
}
const orwell = createElf("Orwell", "bow");
orwell.attack();
const sally = createElf("Sally", "fire");
sally.attack();

如果不要使用 Object.create 呢？ 這時候可以使用 Constructor 函式

//Constructor Functions
function Elf(name, weapon) {
this.name = name;
this.weapon = weapon;
}
Elf.prototype.attack = function () {
return "attack with " + this.weapon;
};
const orwell = new Elf("Orwell", "bow");
console.log(orwell.attack());
//使用 Function constructor
const Elf1 = new Function(
"name",
"weapon",
`this.name = name
this.weapon = weapon`
);
const sally = new Elf1("Sally", "fire");

ES6 新增了 Class 語法糖後便可寫成

class Elf {
constructor(name, weapon) {
this.name = name;
this.weapon = weapon;
}
attack() {
return "attack with " + this.weapon;
}
}
const orwell = new Elf("Orwell", "bow");

繼承 Inheritance#

使用 extends 關鍵字

class Character {
constructor(name, weapon) {
this.name = name;
this.weapon = weapon;
}
attack() {
return "attack with " + this.weapon;
}
}
class Elf extends Character {
constructor(name, weapon, type) {
//如果繼承類別中有建構函式，則需要super將建構函式連接到父類別
//呼叫父類別的constructor函式
//若沒有super，則無法在constructor使用this
super(name, weapon);
this.type = type;
}
}
const dob = new Elf("Dob", "fire", "house");

物件導向特性#

  1. 封裝(encapsulation): 將資料和函式封裝在物件結構中，對資料的操作只能通過已定義的介面，不需要了解細節
  2. 繼承(inheritance) ：子物件可以繼承父物件的所有欄位與屬性，再新增自己的屬性或方法
  3. 多型(polymorphism): 子類別除了擁有自己的方法外，這個方法還能覆寫來特化父類別的同名方法，以賦予其更特殊的行為。
