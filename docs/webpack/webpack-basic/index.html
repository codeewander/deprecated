Webpack 基本設定

Webpack 是什麼？#

  • 靜態模組打包工具(module bundler)
  • 將前端所有資源檔案(包括：js/json/css/img 等)都視作模組處理。Webpack 會根據模組的相依關係進行靜態分析，打包成對應的靜態檔案。

核心要素#

  1. Entry: 指定打包起始點，分析構建內部相依圖
  2. Output: 指定輸入點以及命名輸出資料夾
  3. Loader：讓 Webpack 可以處理非 JS 檔案(Webpack 本身只理解 JS 和 Json)
  4. Plugin: 套件，可執行範圍更大的任務，例如：打包優化、壓縮等
  5. Mode: 指定模式，例如：development、production

基本設定#

webpack 設定檔(基於 Node.js 環境，預設使用 commonjs)

//webpack.config.js
// resolve 用來串接絕對路徑
const { resolve } = require("path");
module.exports = {
//入口
entry: "./src/index.js",
//輸出
output: {
//輸出檔案名
filename: "build.js",
//輸出路徑
//__dirname代表當前檔案的目錄絕對路徑，build為輸出資料夾名稱
path: resolve(__dirname, "build"),
},
module: {
rules: [],
},
plugins: [],
mode: "development",
};

打包 CSS 檔案#

需要 css-loader 和 style-loader

yarn add css-loader style-loader -D
//webpack.config.js
module.exports = {
...,
module: {
rules: [
{
//匹配哪些檔案
test: /\.css$/,
use: [
// 使用哪些 loader 處理(執行順序：從下到上)
//創建style標籤並將JS中的樣式插入，添加到 HTML 的 Head 中就會生效
'style-loader'
//將 CSS 檔案轉成commonjs模組加載到JS中，內容為樣式字串
'css-loader',
]
},
{
test: /\.(scss|sass)$/,
use: [
//sass-loader將檔案編譯成css-loader
'style-loader', 'css-loader', 'sass-loader'
]
}
]
}
};

生成 HTML 檔案#

使用 html-webpack-plugin 套件

yarn add html-webpack-plugin -D
// webpack.config.js
const HtmlWebpackPlugin = require('html-webpack-plugin')
module.exports = {
...,
module: {
plugins:[
//html-webpack-plugin: 預設創建空的Html檔案，自動引入打包輸出的所有資源
new HtmlWebpackPlugin({
//複製此HTML檔案，自動引入打包輸出的所有資源(需要HTML結構內容)
template: './src/index.html'
})
],
}
};

打包圖片檔案#

使用url-loader(相依於 file-loader) 和html-loader

yarn add url-loader file-loader html-loader -D
//webpack.config.js
module.exports = {
...,
module: {
rules:[
{
//匹配格式
//處理圖片，但不會處理 HTML 檔案中的 img 圖片
test: /\.(jpg|png|gif)$/,
//使用一個loader可用loader屬性
loader: 'url-loader',
options: {
//圖片大小小於8kb，就會用base64處理
//優點：減少請求數量和伺服器壓力，缺點：圖片檔案變大，檔案請求速度變慢
limit: 8 * 1024,
//由於url-loader預設使用ES6解析，而html-loader使用common.js引入圖片，因此解析時會有問題
//需要關閉url-loader的ES5解析，使用common.js解析
esModule: false,
//圖片命名([hash:10]指取hash前10位數字，[ext]指取檔案原來檔名格式)
name: '[hash:10].[ext]',
//輸出檔案放到 images 資料夾下
outputPath: 'images'
}
},
{
test: /\.html$/,
//處理Html檔案中的img圖片(負責引入img，才能被url-loader處理)
loader: 'html-loader'
}
]
}
};

打包其他資源#

//webpack.config.js
module.exports = {
...,
module: {
rules:[
{
//打包其他資源(除特定檔案格式外)
exclude: /\.(css|js|html｜jpg|png|gif)/,
loader: 'file-loader',
options: {
name:'[hash:10].[ext]',
//輸出檔案放到media資料夾下
outputPath: 'media'
}
}
]
}
};

devServer 本地伺服器#

支援 Live Reload，程式碼有變動就會自動編譯、自動打開瀏覽器、自動更新瀏覽器頁面 特性：只會在記憶體中編譯打包，不會有任何輸出

yarn add webpack-dev-server -D
module.exports = {
...,
devServer: {
contentBase: resolve(__dirname, 'build'),
//啟動gzip壓縮
compress: true
//設定port號
port: 3000,
//自動打開瀏覽器
open: true
}
};

修改 package.json 檔案

{
"scripts": {
"dev": "webpack-dev-server --mode development",
"build": "webpack --mode development"
}
}

啟動 devServer 指令

yarn run dev

將 CSS 樣式提取成為單一檔案#

使用css-loader和style-loader打包後，樣式會在 JS 檔案中，倘若想要將 CSS 樣式提取出來成為單一檔案，則需要使用 mini-css-extract-plugin套件。

module.exports = {
...,
module: {
rules: [
{
test: /\.css$/,
use: [
//取代 style-loader(因為style-loader會將樣式輸入至js)
//'style-loader'
MiniCssExtractPlugin.loader,
'css-loader',
]
},
]
},
plugin: [
new HtmlWebpackPlugin({
template: './src/index.html'
}),
new MiniCssExtractPlugin({
//輸出CSS檔案名
filename: 'css/style.css'
})
]
};

CSS 兼容性處理#

使用 postcss-loader 和 postcss-preset-env

yarn add postcss-loader postcss-preset-env -D
//webpack.config.js
const MiniCssExtractPlugin = require('mini-css-extract-plugin')
//設定Node環境變數
process.env.NODE_ENV = 'development'
module.exports = {
...,
module: {
rules: [
{
test: /\.css$/,
use: [
MiniCssExtractPlugin.loader,
'css-loader',
{
//postcss-preset-env: 透過package.json中browserslist的設定加載指定的CSS兼容性樣式(預設看production環境，需透過設定Node環境變數來改變，與config中的mode屬性無關)
loader : 'postcss-loader',
options:{
ident: 'postcss',
//postcss的套件
plugins: ()=> [
require('postcss-preset-env')()
]
}
}
]
},
]
},
};
//package.json
{
browserslist :{
//開發環境
"development":[
"last 1 chrome version",
"last 1 firefox version",
"last 1 safari version"
],
//生產環境
"production": [
">0.2%",
"not dead"
]
}
}

壓縮 CSS#

使用optimize-css-assets-webpack-plugin

yarn add optimize-css-assets-webpack-plugin -D
const OptimizeCssAssetsWebpackPlugin = require('optimize-css-assets-webpack-plugin')
module.exports = {
...,
module: {
rules: [...]
},
//要注意套件順序
plugin: [
new HtmlWebpackPlugin({
template: './src/index.html'
}),
new MiniCssExtractPlugin({
filename: 'css/style.css'
}),
//壓縮 CSS
new OptimizeCssAssetsWebpackPlugin()
]
};

語法檢查 eslint#

使用 eslint-loader(相依於eslint)

yarn add eslint eslint-loader -D
//webpack.config.js
module.exports = {
...,
module: {
rules: [
{
//只檢查自己寫的程式碼，不檢查第三方套件，因此要排除node_modules裡面的檔案
test: /\.js$/,
exclude: /node_modules/,
//倘若有其他loader也處理js檔案，則此loader優先執行
enforce:'pre',
loader: 'eslint-loader',
options: {
//自動修復錯誤
fix: true
}
},
]
},
};
//package.json 設定檢查規則
{
"eslintConfig": {
//檢查規則
"extends": ...
}
}

兼容性處理 babel#

使用babel-loader

yarn add babel babel-loader @babel/core @babel/preset-env core-js -D
//webpack.config.js
module.exports = {
...,
module: {
rules: [
// 兼容性處理
{
test: /\.js$/,
exclude: /node_modules/,
loader: 'babel-loader',
options: {
//@babel/preset-env 只能轉換基本語法，如promise無法轉換
//另使用 core-js （根據需求進行兼容性處理）
presets: ['@babel/preset-env',{
//按需要加載
useBuiltIns:'usage',
//指令core-js版本
corejs:{
version: 3
},
//指定瀏覽器兼容性問題
targets:{
chrome: '60',
firefox: '60'
}
}]
}
}
]
},
};

壓縮 HTML 檔案#

//webpack.config.js
module.exports = {

new HtmlWebpackPlugin({

//HTML壓縮

]
};
