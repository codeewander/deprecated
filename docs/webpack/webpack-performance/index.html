Webpack 性能優化

可分為兩個不同階段的性能優化,分別是開發階段(development)和生產階段(production)。

開發階段(Development)#

優化打包速度#

Hot Module Replacement(HMR)#

作用：一個模組有改變，只會重新打包該模組，而不是打包所有模組

//webpack.config.js
module.exports = {
...,
//入口: 加入html檔案
entry: ['./src/index.js','./src/index.html'],
module: {
rules: [
{
test: /\.css$/,
use: [
'style-loader'
'css-loader',
]
},
]
}
devServer: {
...
//開啟HMR功能
// 樣式檔案: 預設可使用HMR功能(修改style，JS不會重新打包，style-loader內部實現了)
// JS 檔案: 預設無法使用HMR
// ->解決方法：：修改js檔案，增加支援HMR功能的程式碼(只能處理非入口檔案的js檔案)
// HTML檔案：預設無法使用HMR，且 html 無法hot reload (只有一個html，不需要HMR)
// ->解決方式：修改entry入口，將html檔案引入
hot: true
}
};
// 一旦 module.hot為 true，表示HMR為true
if(module.hot){
//監聽test.js檔案變化，一旦發生變化則執行後面callback，其他模組不會重新打包
module.hot.accept('./test/js', function (){
})
}

優化程式碼測試#

source-map#

作用：

  • 提供原始程式碼和打包後程式碼之間相對應的檔案，打包後程式碼出錯可以透過此檔案快速追蹤找到原始程式碼的錯誤
  • 瀏覽器可以直接看到打包前的程式碼
//webpack.config.js
module.exports = {
...,
//入口: 加入html檔案
entry: [...],
module: {},
devServer: {
hot: true
},
//開啟source-map工具
//設定值[inline-|hidden-|eval-][nosources-][cheap-[module-]]source-map
devtool: 'source-map'
};
  • 開發環境：速度快，容易找錯誤 => eval-source-map /eval-cheap-module-source-map
  • 生產環境：原始程式碼隱藏，容易找錯誤 => nosources-source-map 全部隱藏/ hidden-source-map 隱藏原始程式碼

sourcemap 詳細設定值@webpack

生產階段(Production)#

優化打包建構速度#

oneOf#

作用：在 oneOf 內的 loader 只會匹配一次，不用全部的 loader 都遍歷，可以優化打包速度

module.exports = {
// 忽略前面配置
rules:[
// 因為eslint-loader和babel-loader都會處理 js 檔案
// 因此，先把eslint-loader單獨從 oneOf 拿出来，如此，才會兩個loader都執行
{
test:/\.js$/,
exclude:/node_modules/,
loader:'eslint-loader',
enforce:'pre',
options:{
fix:true
}
},
{
//oneOf 內的loader只會匹配一個
oneOf:[
{
test:/\.js$/,
exclude:/node_modules/,
loader:'babel-loader',
options:{
// ...
}
]
}
]
}

babel 緩存#

//webpack.config.js
module.exports = {
...,
module: {
rules: [
// 兼容性處理
{
test: /\.js$/,
exclude: /node_modules/,
loader: 'babel-loader',
options: {
presets: [...] ,
//開啟 babel 緩存
//第二次打包時會讀取緩存
cacheDirectory: true
}
}
]
},
};

優化程式碼運行速度#

文件資源緩存： 檔名 hash#

每次 webpack 打包時都會產生唯一的 hash 值，然而由於 js 和 css 同時使用同個 hash 值，導致改動一個檔案重新打包，所有緩存都會失效 => 可以改成使用 contenthash，倘若檔案內容不同，則 hash 值不同。

module.exports = {
entry: './src/index.js',
output:{
//輸出檔案名
filename:'js/build.[contenthash:10].js',
path: resolve(__dirname, 'build')
},
plugin: [
...
new MiniCssExtractPlugin({
filename: 'css/style.[contenthash:10].css'
}),
...
]
};

tree shaking#

作用：去除無用的程式碼 條件：1. 必須使用 ES6 模組 2. production 環境

在 package.json 加上 sideEffects 屬性並設定為 false，可以讓 Webpack 在打包時檢查是否有沒有作用的程式碼，這些程式碼不要打包輸出至檔案中

{
//有時候會把一些檔案移除如：CSS、@babel/polyfill 等，因此要把這些檔案加入排除
"sideEffects": ["*.css"]
}

code split#

倘若多個文件都引入同一個套件，在 webpack 打包時會重複將此套件包進這三個檔案中，若要將共用套件抽出來，需要使用 optimization屬性的 splitChunk

const {resolve} = require('path')
const HtmlWebpackPlugin = require（'html-webpack-plugin'）
module.exports = {
//單入口
entry: 's./src/js/index.js',
//多入口
// entry :{
// main: './src/js/index.js',
// test: './src/js/test/js'
// },
output:{
//使用 [name] 取文件名
//chunkFileName為共用檔案名稱
filename:'[name].[contenthash:10].js',
chunkFileName: '[name].shared.js',
path: resolve(__dirname, 'build')
},
plugin: [
new HtmlWebpackPlugin({
template: './src/index/html',
minify:{
collapseWhitespace: true,
removeComments: true
}
})
],
//將node_modules的程式碼單獨打包成一個chunk輸出
//自動分析多入口chunk是否有公用檔案，若有則會打包成單獨的chunk
optimization: {
splitChunks:{
chunks: 'all'
}
},
mode:'production'
};

lazy loading/prefetch JS 延遲加載/預加載#

lazy loading 作用：執行某程式時，才將某些程式碼載入，減少沒有被執行的程式碼仍被載入的資源浪費

原本的寫法，即使沒有按下按鈕，仍然會被載入 add，只是沒有被調用

import { add } from "./add";
document.getElementById("btn").onclick = function () {
console.log(add(1, 2));
};

可以使用 lazy loading 方式優化如下：

document.getElementById("btn").onclick = function () {
import(/*webpackChunkName:'add'*/ "./add").then(({ add }) => {
console.log(add(1, 2));
});
};
  • import().then() 是動態載入函式的方法
  • /* webpackChunkName: login */ 是當 Webpack 在打包時會將這一段 function 抽離成單一一個 .js 檔

lazy loading 的前提是要進行分割程式碼

倘若希望使用之前，提前加載 JS 檔案可加上 prefetch

正常加載屬於並行加載(同一時間加載多個檔案)，prefetch 則是等瀏覽器將其他資源加載完有空，再偷偷加載資源。

要注意的是：prefetch 兼容性不高

document.getElementById("btn").onclick = function () {
import(/*webpackChunkName:'add', webpackPrefetch: true*/ "./add").then(
({ add }) => {
console.log(add(1, 2));
}
);
};
